# R-Shell Custom — лёгкий SSH/SFTP-клиент для macOS и Linux

[![Tauri](https://img.shields.io/badge/Tauri-2.0-blue?logo=tauri)](https://tauri.app/)
[![React](https://img.shields.io/badge/React-19-61dafb?logo=react)](https://react.dev/)
[![Rust](https://img.shields.io/badge/Rust-orange?logo=rust)](https://www.rust-lang.org/)
[![Release](https://img.shields.io/github/v/release/DeveloperrEva/r-shell-custom?label=release)](https://github.com/DeveloperrEva/r-shell-custom/releases/latest)

Современный SSH / SFTP / FTP / RDP / VNC клиент на Rust + Tauri 2. **Нативная скорость, ~34 МБ памяти** (в ~50 раз меньше Java-клиентов), установщик меньше 10 МБ.

Это **пользовательский форк** [GOODBOY008/r-shell](https://github.com/GOODBOY008/r-shell) с рядом доработок и локализацией на русский. Сделан для себя и для ру-комьюнити.

## ✨ Чем отличается от оригинала

Всё перечисленное добавлено/изменено поверх апстрима [GOODBOY008/r-shell](https://github.com/GOODBOY008/r-shell) 2.5.0:

| Что | Описание |
|-----|----------|
| 🇷🇺 **Русский язык** | Полная локализация интерфейса (929+ ключей, паритет с en/zh). Переключается в **Настройки → Язык → Русский**. |
| 📂 **История директорий SFTP + закрепление** | Кнопка 🕘 в панели файлов: список недавних папок и закреплённые (pin) — **отдельно для каждого сервера**. Быстрый переход одним кликом, переживает перезапуск. |
| 🖥️ **Локальный терминал с вкладками** | Кнопка «New Local Terminal» — запускает локальную оболочку (`$SHELL`) на вашей машине, много вкладок. Правая/нижняя панели скрываются, т.к. сервера нет. |
| 🧩 **Перенос серверов между группами из UI** | Пункт контекстного меню «Переместить в группу». |
| 📝 **Открытие SFTP-файлов в VS Code** | Двойной клик по удалённому файлу открывает его в VS Code; при сохранении файл автоматически заливается обратно на сервер. |
| ⚡ **Плавный ввод в терминале** | Убрана задержка эха: `TCP_NODELAY` на всех сокетах + синхронный fast-path отрисовки + меньший интервал флуша. Печатается так же плавно, как в нативном терминале. |
| 🔕 **Убран GPU-мониторинг** | Полностью вырезан (фронтенд + Rust + i18n) — меньше кода и нагрузки. |
| 🔄 **Автообновление со своего репозитория** | Апдейтер апстрима удалён (он затирал форк), включён свой — обновления прилетают из [релизов этого репозитория](https://github.com/DeveloperrEva/r-shell-custom/releases), подписанные. |
| 🍎🐧 **Только macOS и Linux** | Сборки и упоминания Windows убраны — форк ориентирован на macOS (Apple Silicon) и Linux. |

Полный список правок с деталями реализации — в [`FORK_CHANGES.md`](FORK_CHANGES.md).

## 📦 Установка

Скачайте с [страницы релизов](https://github.com/DeveloperrEva/r-shell-custom/releases/latest):

| Платформа | Файл |
|-----------|------|
| **macOS (Apple Silicon, M-чипы)** | `r-shell_x.y.z_aarch64.dmg` |
| **Linux** | `.AppImage` / `.deb` |

### macOS
Откройте DMG, перетащите **r-shell** в «Программы».

Приложение подписано ad-hoc (без нотаризации Apple), поэтому при первом запуске Gatekeeper может ругаться. Варианты:
- ПКМ по приложению → **Открыть** → **Открыть**; либо
- в терминале: `xattr -dr com.apple.quarantine /Applications/r-shell.app`

### Linux
`.AppImage` — сделайте исполняемым (`chmod +x`) и запустите; либо установите `.deb` (`sudo dpkg -i`).

## 🔄 Автообновление

Приложение при запуске проверяет [релизы этого репозитория](https://github.com/DeveloperrEva/r-shell-custom/releases) и предлагает установить новую версию. Обновления подписаны ключом — посторонний бинарь установить нельзя. Никаких обращений к оригинальному репозиторию (который раньше затирал форк) больше нет.

## ✨ Возможности

- **Мультипротокол** — SSH, SFTP, FTP, RDP, VNC
- **Интерактивный терминал** — полноценный PTY (vim, top, tmux и т.п.), разделение экрана, много вкладок, поиск, локальный терминал
- **Файловый менеджер SFTP/FTP** — загрузка/скачивание, drag-and-drop, права, история директорий + закрепление, открытие в VS Code
- **Удалённый рабочий стол** — RDP и VNC
- **Системный монитор** — CPU, память, диск, сеть, процессы удалённого хоста
- **Просмотр логов** — потоковое чтение и фильтрация
- **Синхронизация каталогов** между локальной и удалённой машиной
- **Внешний вид** — темы терминала, фоновые изображения, прозрачность
- **Низкое потребление памяти** — нативный Rust + Tauri 2, ~34 МБ вместо ~1.7 ГБ у Java-аналогов

## 🛠️ Сборка

### Требования
- Node.js ≥ 18, **pnpm** 9.15.4 (`corepack enable && corepack prepare pnpm@9.15.4 --activate`)
- Rust (stable) через rustup

### macOS (Apple Silicon)
```bash
pnpm install
pnpm tauri build --target aarch64-apple-darwin
```
Артефакты: `src-tauri/target/aarch64-apple-darwin/release/bundle/{dmg,macos}/`.

> ⚠️ На этой машине Node работает под Rosetta (x64), поэтому **обязателен** `--target aarch64-apple-darwin` — иначе Tauri назовёт DMG `_x64`, хотя бинарь arm64.

### Linux
Собирается на Linux-машине или в CI (кросс-компиляция с macOS невозможна — нужны системные библиотеки webkit2gtk):
```bash
pnpm install
pnpm tauri build
```

### Проверки
```bash
pnpm test          # юнит-тесты (vitest)
pnpm i18n:check    # паритет ключей всех локалей
```
