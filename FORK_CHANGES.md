# r-shell — форк (изменения)

Собственная сборка на базе [GOODBOY008/r-shell](https://github.com/GOODBOY008/r-shell) 2.5.0.
Bundle id оставлен прежним (`com.aiden.r-shell`) — новая сборка заменяет установленную и видит те же данные (серверы/группы) в `~/Library/WebKit/com.aiden.r-shell`.

## Что изменено

### 1. Перенос серверов в группы через интерфейс

- Drag-and-drop переноса сервера на папку уже работал в апстриме — оставлен как есть.

- Модель: сервер привязан к группе через поле `folder` = полный путь папки (напр. `All Connections/Work`).

### 2. Удалён GPU-мониторинг (полностью)
- Backend: убраны команды `detect_gpu`/`get_gpu_stats` и вся GPU-секция в `commands.rs`; снята регистрация в `lib.rs`.

### 3. Открытие SFTP-файла в VS Code с обратной синхронизацией
- **Двойной клик** по файлу открывает его сразу в **VS Code** (round-trip). Встроенный редактор остался доступен через пункты меню «Open»/«Edit».
- В контекстное меню файла добавлен пункт **«Открыть в VS Code»**.
- Поток: файл скачивается во временную папку → открывается в VS Code → фоновая задача опрашивает файл (раз в ~1 с) и при **каждом сохранении** заливает изменения обратно на сервер. Тост «Сохранено на сервер» при успехе.
- Новый Rust-модуль `src-tauri/src/editor/mod.rs` (команды `open_in_external_editor`, `close_external_editor_session`, `list_external_editor_sessions`), регистрация в `lib.rs`.
- Frontend: `integrated-file-browser.tsx` (пункт меню + обработчик), `App.tsx` (слушатель событий `editor-sync`/`editor-sync-error`), i18n-ключи.
- Поиск бинарника VS Code: CLI `/Applications/Visual Studio Code.app/.../bin/code` → `code` в PATH → `open -a "Visual Studio Code"`.
- Особенности реализации (после адверсариального ревью): атомарное резервирование сессии под локом (нет гонок при двойном клике); уникальная временная папка на сессию; дедуп ошибок заливки + backoff (нет спама тостов при недоступном сервере); очистка временных папок на ошибках и подметание при старте приложения.
- Синхронизация работает, пока открыто приложение; старые временные папки убираются при следующем запуске.

### 4. Локальный терминал с вкладками
- Кнопка **«New Local Terminal»** (иконка терминала) в баре вкладок — открывает вкладку с **локальной оболочкой** (спавн `$SHELL`/zsh на этой машине), не по SSH.
- **Много вкладок**: каждая новая локальная вкладка = свой `crypto.randomUUID()` = свой отдельный процесс-оболочка. Полная изоляция (демультиплексинг вывода по `connection_id`), поддержка resize, reconnect, IME — переиспользуется существующий xterm+WebSocket+PTY-конвейер.
- Реализация: новый Rust-модуль `src-tauri/src/local_pty.rs` (через крейт `portable-pty`, спавн shell + мост блокирующих read/write к tokio-каналам, как у SSH). Маршрутизация — флаг `local: bool` в WS-сообщении `StartPty` (`websocket_server.rs`) → `ConnectionManager::start_local_pty_connection`. `PtySession` стал транспортно-нейтральным (убрано мёртвое SSH-поле `channel_id`). Frontend: проп `isLocal` у `PtyTerminal`, `handleNewLocalTerminal` в `App.tsx`.
- Локальные вкладки эфемерны (не сохраняются между запусками — при рестарте не остаётся «висящих» плейсхолдеров). Оболочка спавнится лениво при старте PTY; закрытие вкладки/приложения убивает процесс.
- **Когда активна локальная вкладка**, скрываются правая панель (монитор + логи) и нижняя (файловый менеджер) — у локального терминала нет удалённого сервера. При переключении на SSH/SFTP-вкладку панели возвращаются в прежнем виде (через существующий флаг `hideExtraPanels` в `App.tsx`, персистентность раскладки не трогается).

### 5. Исправлен лаг ввода в терминале (TCP_NODELAY)
- Симптом: при SSH-подключении набор текста в терминале «подлагивал», хотя обычный `ssh` в Терминале работал плавно.
- Причина: библиотека `russh` открывает TCP-сокет **не отключая алгоритм Нейгла** — маленький пакет с нажатием клавиши придерживается до ACK предыдущего (до ~40–200 мс). OpenSSH для интерактивных сессий всегда ставит `TCP_NODELAY`. Эффект усиливался фоновыми запросами монитора/логов по тому же соединению.
- Фикс: сокет создаётся вручную с `set_nodelay(true)` и передаётся в `client::connect_stream` — в `src-tauri/src/ssh/mod.rs` (SSH/PTY — основной фикс) и `src-tauri/src/sftp_client.rs` (standalone SFTP). Плюс `TCP_NODELAY` на принятом локальном WebSocket-сокете в `websocket_server.rs` (по нему идёт каждый кадр вывода в xterm).

### 5. Задержка эха в терминале (плавность набора)
Набор в терминале «подлагивал» (эхо символов появлялось с задержкой), хотя в нативном Терминале macOS всё плавно. Причина — накопление мелких задержек на каждом символе. Исправлено на трёх уровнях:
- **TCP_NODELAY** на всех сокетах (`ssh/mod.rs`, `sftp_client.rs`, `websocket_server.rs`) — отключён алгоритм Нейгла, который придерживал каждый маленький пакет эха до подтверждения предыдущего (russh сам его не выставляет; сделан ручной `connect_stream` с `set_nodelay(true)`). Главная причина.
- **Frontend fast-path** (`pty-terminal.tsx`): мелкий вывод (эхо клавиши, ≤2 КБ) при простое пишется в xterm **синхронно**, минуя кадр `requestAnimationFrame` (−8…16 мс на символ). Пакетная запись через RAF сохранена для потока; защита от OOM держится на инварианте «не более одной синхронной записи за кадр» (`rafId===null`), вся логика кредитов/watermark/лимита вынесена в общий `writeChunk`.
- **Backend flush** (`websocket_server.rs`): `OUTPUT_FLUSH_INTERVAL_MS` снижен 10→3 мс — убирает удержание 2-го+ символа при быстром наборе; флуд по-прежнему ограничен размерным гейтом 16 КБ, не интервалом.
- Диагностика и правки прошли через воркфлоу с адверсариальной верификацией каждой правки на регресс защиты от флуда/OOM/backpressure и работу локального терминала.

### 6. Автообновление — со своего репозитория
Сначала апдейтер апстрима был удалён (он указывал на релизы `GOODBOY008/r-shell` и затирал форк — установленная 2.3.0 однажды сама обновилась до апстрим-2.5.0). После создания **собственного** репозитория (`DeveloperrEva/r-shell-custom`) автообновление включено обратно, но нацелено на **свои** релизы — это безопасно.
- `tauri.conf.json` — `plugins.updater.endpoints` = `https://github.com/DeveloperrEva/r-shell-custom/releases/latest/download/latest.json`; `bundle.createUpdaterArtifacts=true`; `pubkey` = наш публичный ключ подписи.
- Ключ подписи: `~/.tauri/r-shell-custom.key` (**вне репозитория**, в git не коммитится). Сборка подписывает `latest.json`/DMG через env `TAURI_SIGNING_PRIVATE_KEY`.
- `update-checker.tsx` восстановлен: авто-проверка при старте + диалог загрузки/установки/перезапуска (плагины `@tauri-apps/plugin-updater` + `plugin-process`).
- **ВАЖНО:** endpoint должен указывать ТОЛЬКО на наш репозиторий, никогда на апстрим. Публичный репо нужен, чтобы ассеты релизов качались без токена.
- Как выпускать обновление: поднять `version` в `tauri.conf.json`/`Cargo.toml`/`package.json`, собрать (`TAURI_SIGNING_PRIVATE_KEY=$(cat ~/.tauri/r-shell-custom.key) pnpm tauri build --target aarch64-apple-darwin`), создать GitHub Release с тегом `vX.Y.Z` и залить `*.dmg`, `*.app.tar.gz`, `*.app.tar.gz.sig`, `latest.json`.

### 8. История директорий SFTP + закрепление (per-server)
Быстрый переход по посещённым директориям и закрепление часто нужных — своё для каждого сервера.
- Новый модуль `src/lib/sftp-directory-history.ts` — хранилище в localStorage (`r-shell-sftp-directory-history`), ключ = **сервер** (`connectionId`, с отбрасыванием суффикса `-dup-<ts>` у дублированных вкладок, чтобы вкладки одного сервера делили историю). На сервер: список недавних (до 30, дедуп, свежие сверху) + список закреплённых (до 100). История одного сервера никогда не видна на другом.
- UI в тулбаре SFTP (`integrated-file-browser.tsx`): кнопка 🕘 (History) открывает Popover с секциями «Закреплённые» и «Недавние» (клик = переход; закрепить/открепить/убрать; «Очистить»; «Закрепить текущую»).
- Запись в историю — при успешной загрузке директории (после проверки на устаревший запрос). Дубли-вкладки и удаление сервера обрабатываются: `forgetServer(id)` вызывается из `handleDelete` в `connection-manager.tsx`, очищая историю удалённого сервера.
- i18n: блок `fileBrowser.history.*` + тост `fileBrowser.toast.dirGone/dirGoneDesc` (en/ru/zh, паритет 941 ключ).
- Прошло адверсариальное ревью (3 измерения, 8 находок): исправлены (1) запись пути прошлого сервора при закреплении во время загрузки — кнопка «Закрепить текущую» блокируется на время загрузки; (2) устаревший `dirState` на 1 кадр после смены сервера — синхронный пересчёт в рендере; (3) неограниченный рост хранилища — `forgetServer` + лимит закреплённых; (4) тихий редирект на /home для мёртвого pin — теперь тост. Юнит-тесты: `src/__tests__/sftp-directory-history.test.ts` (19 passed).
